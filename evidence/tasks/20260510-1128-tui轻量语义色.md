# TUI 轻量语义色

## 背景

用户确认采用轻量语义色方案，希望 TUI 在保持 Codex/zread 式克制命令感的基础上增加层次，不要过度夸张；完整 JSON 仍需保持干净可复制。

## 本次变更

- 为现代 TUI 增加集中式 ANSI 语义色 helper，避免颜色码散落在业务逻辑中。
- `kc ›` prompt 改为青色加粗。
- 启动标题使用青色，普通提示使用灰色，token 缺失与默认预算提示使用黄色。
- 状态栏使用深色底、浅灰辅助信息；`auth:missing` 使用黄色，已配置认证使用绿色。
- 补全菜单采用深色背景，命令/当前项有更清晰的对比，说明信息使用低饱和灰。
- 命令转录中 `$` 使用灰色，`kc` 使用青色。
- 结果摘要中 `[command] OK` 使用绿色，`ERROR` 使用红色，预算/Token/确认类提示使用黄色。
- `/json` 输出仍直接打印 `json.dumps(...)`，不插入 ANSI 颜色码。
- 根据截图反馈修正底部状态栏：整条 toolbar 统一使用 `#111315` 深色背景，避免剩余区域出现突兀白色反显。
- 将结果后的 JSON 提示从较重的完整文案收敛为低对比 `json: /json`。
- 启动提示改为 `Type / for commands. Ctrl-L clears.`，避免重复解释 `/json`。
- 修正兜底摘要中过时的 “JSON block” 文案，保持和当前按需 `/json` 行为一致。

## 后续建议

- 后续若继续调整视觉，应优先通过真实终端截图确认对比度，再微调 prompt_toolkit style 字典。
- 如果未来支持浅色终端主题，可增加 `KEEPA_CLI_THEME=dark|light|auto`，但当前先保持单一深色友好主题，避免配置膨胀。
