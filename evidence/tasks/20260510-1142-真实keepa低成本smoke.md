## 2026-05-10 继续优化：完整产品详情预设

用户指出应参考 `D:\ProductSelectionSystem` 的 Keepa 请求方式，让 1 token 能尽量拿到完整产品信息。对照发现该项目产品请求默认使用：

- `history=1`
- `stats=180`
- `videos=1`
- `aplus=1`
- 可选 `days`

本项目此前 service 层已支持多数字段，但 CLI 只暴露 `history / stats / update / offers`。本轮补齐：

- `products get --full`：低成本完整详情预设，自动发送 `history=1`、`stats=180`、`videos=1`、`aplus=1`。
- 显式参数：`--days`、`--videos`、`--aplus`、`--rating`、`--buybox`、`--stock`、`--code-limit`、`--only-live-offers`、`--historical-variations`。
- `products get --out <file>`：完整响应很大时把 body 写入 JSON 文件，终端 envelope 仍保留 output 元数据。
- 保守成本边界：`--full` 不自动开启 `offers`、`stock` 等可能显著增加 token 成本的参数。

## 2026-05-10 继续优化：Agent 产品视图

用户指出 `history` 字段与其他大字段最终要接入 Agent，不应让 Agent 直接消费 Keepa 原始 `csv` / `stats.current` 位置数组。本轮做输出契约优化，默认 raw 输出不变，新增显式 Agent 视图：

- `products get --agent-view` / `--view agent`：把 Product Object 转为稳定摘要。
- `--history-limit <n>`：控制每个历史序列在 stdout 中保留的最近点数。
- `--out <file>` 与 Agent 视图可同时使用：文件保存完整 raw body，stdout 仅返回摘要与 `data.raw.output.path`。
- 新增 `keepa_cli/product_view.py`，避免继续扩大 `service.py`。

### Agent 视图字段

- `identity`：ASIN、标题、品牌、制造商、domain、产品类型、urlSlug、parentAsin 等。
- `category`：root category、category tree、sales rank reference、current rank、salesRanks 摘要。
- `pricing`：current/avg30/avg90/avg180 价格、Buy Box、coupon、promotions。
- `demand`：monthlySold、monthlySoldHistory、salesRankDrops 30/90/180/365。
- `rating`：rating 与 review count，保留 raw value。
- `offers`：total/retrieved/FBA/FBM offer count、buyBoxEligibleOfferCounts、缺货百分比。
- `media` 与 `aplus`：图片、视频、A+ block/module/image/text 样本。
- `content`、`compliance_and_logistics`、`variations`：Agent 做筛选、报告和审计时的常用字段。
- `stats_summary` 与 `history_summary`：按 Keepa 官方 CsvType index 命名，省略完整 raw 数组。
- `raw_field_presence` 与 `raw`：标记原始字段存在性，并指向完整 raw 输出文件或 fixture。
