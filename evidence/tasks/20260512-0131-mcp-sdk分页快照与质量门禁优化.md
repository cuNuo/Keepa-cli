## 任务目标

- 继续优化隔离官方 Python MCP SDK adapter，不替换生产 `python -m keepa_cli --mcp` 入口。
- 将 SDK typed `resources/list`、`resources/templates/list` 与 `prompts/list` 改为压缩首页 + cursor 全量发现，降低 Agent 起手上下文。
- 增加可复现 Inspector/SDK typed snapshot，减少手工 UI 复核依赖。
- 增加统一 MCP quality gate，并接入 release gate 与远端 `mcp-sdk-adapter` CI job。
- 补齐测试、文档、evidence、提交推送与远端 CI 验证。

## 背景与输入

- 用户要求继续全面优化 MCP 实现，并强调 SDK adapter 仍应隔离 spike，不直接替换生产 stdio 入口。
- 上一轮已完成官方 low-level `Server("keepa_mcp")` adapter、`.[mcp-sdk]` CI job、typed fixture 映射、HTTP boundary fixture 与产品 live read 手动流程。
- 官方 Python MCP SDK typed `ClientSession.list_*()` 当前只支持标准 `cursor`，不支持 Keepa 扩展 `toolset/limit`；因此本轮继续通过 adapter 层起手排序和分页控制上下文。

## 处理过程

### 1. SDK typed 起手分页

- `keepa_cli/agent/mcp_sdk_adapter.py` 新增 SDK typed 首页大小：
  - tools：8。
  - resources：6。
  - resource templates：6。
  - prompts：4。
- `tools/list` 继续保留 `toolset=all` 全集，第一页以 `keepa.context_policy` 起手。
- `resources/list` 首页以 `keepa://context/policy` 起手。
- `resources/templates/list` 首页以 `keepa://toolsets/{toolset}` 起手。
- `prompts/list` 首页以 `keepa.product_research` 起手。
- cursor payload 增加 `sdk_collection`，防止跨集合误用 cursor；旧 payload 只含 `sdk_offset` 时仍兼容读取。

### 2. typed fixture 与 smoke 扩展

- `scripts/smoke_mcp_sdk_adapter_client.py` 现在会按 cursor 收齐 tools、resources、resource templates 与 prompts，并验证各集合首页大小和优先项。
- `scripts/check_mcp_sdk_adapter_typed_fixture.py` 继续把 Inspector fixture 映射为官方 SDK typed client 调用，同时记录 resources/templates/prompts 的 `page_count`、首页元数据和分页约束。
- `tests/test_mcp_sdk_adapter.py` 扩展 SDK smoke、typed fixture、adapter status 断言。

### 3. Inspector snapshot 与统一门禁

- 新增 `scripts/export_mcp_inspector_snapshot.py`：
  - 通过官方 SDK `ClientSession` 连接 `python -m keepa_cli.agent.mcp_sdk_adapter --stdio`。
  - 记录 `initialize`、tools/resources/templates/prompts 首页与分页、`keepa://context/policy` 资源读取、非法工具调用错误映射和 `ping`。
  - `--check` 模式只校验形状和关键协议字段，不写入 runtime 工件。
- 新增 `scripts/check_mcp_quality_gate.py`：
  - 聚合 `check_agent_eval_fixtures.py`、`compare_mcp_sdk_adapter_fixture.py`、SDK smoke、typed fixture 和 snapshot check。
  - `--require-sdk` 用于 CI 强制确认 `.[mcp-sdk]` 已安装；`--skip-if-missing` 用于普通本地 release gate 降级。
- `scripts/release_gate.py` 改为调用统一 MCP quality gate。
- `.github/workflows/ci.yml` 的 `mcp-sdk-adapter` job 改为安装 `.[mcp-sdk]` 后运行 SDK 单测与 `scripts/check_mcp_quality_gate.py --require-sdk`。

### 4. 文档同步

- README、README.zh-CN、`docs/agent-contract.md`、`docs/architecture/mcp-python-sdk-adapter-comparison.md`、`docs/index.html` 已同步 SDK typed 分页、snapshot 与 quality gate 说明。
