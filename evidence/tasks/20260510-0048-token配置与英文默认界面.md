## 任务目标

- 精简现代 TUI 首屏文案，避免过度说明。
- 补齐 Keepa token 配置入口，让 TUI/CLI/doctor/live client 形成真实闭环。

## 背景与输入

- 用户反馈：TUI 文案过多；当前没有登录/配置 token 的地方；README 应参考大项目写法，包含居中介绍、标签、配置说明。
- 用户追加要求：默认英文，然后可以选择中文语言。
- 偏差说明：项目全局规则要求用户可见文档默认中文；本轮按用户显式要求将 README 与默认 TUI 文案改为英文，并通过 `config set-language zh` 与 `README.zh-CN.md` 提供中文选项。

## 处理过程

- 配置层：新增 `api_key` 持久化写入、`language` 默认值 `en`、`config set-token`、`config set-language`。
- 认证层：`doctor` 支持识别 config token；环境变量 `KEEPA_API_KEY` 优先于 config。
- client 层：live request 可从 `KEEPA_CLI_CONFIG` 指向的配置文件读取 token，输出仍不泄露明文。
- TUI：默认英文短文案；增加 token 输入框与保存按钮；读取 config 的 `language=zh` 时切换中文。
- README：英文默认首页，含居中标题、徽章、导航、安装、token 配置、语言、TUI、Agent Mode、开发与安全；新增 `README.zh-CN.md`。
