## 追加记录：TUI 命令输入与预算配置修正

### 任务时间

- 更新时间：2026-05-10
- 当前状态：已完成

### 背景与输入

- 用户反馈：配置过 token 和 max tokens 后不应继续占据大块 TUI 空间；初始焦点应在命令输入；命令输入无效且结果只显示原命令；输出不便复制；输入 `/` 时应像 Codex 一样弹出相关命令。
- 用户要求参考 `cli-creator`：CLI 要有明确 auth/config、稳定 JSON、doctor、能力发现、命令补全和可审计输出。
- 外部依据：公开 Keepa Python API 文档将 access key 描述为 64 character access key string；本轮只做本地形态校验，不承诺服务端有效性。

### 处理过程

- 配置校验：`config set-token` 保存前校验 token 必须为 64 个可见 ASCII 字符、无空白；错误不会写入配置文件。
- 预算配置：新增 `config set-max-tokens`，写入 `max_tokens_per_request`，用于高订阅场景调大单次请求 token 预算提示；接入 service、CLI、capabilities 和 schema snapshot。
- TUI 布局：现代 TUI 改为命令输入优先；启动后焦点固定到 `#command-input`；token/预算输入只在配置不完整时显示，配置完成后折叠为状态行摘要。
- TUI 命令体验：输入 `/` 或前缀时在命令输入上方显示匹配命令；执行后标题显示解析后的 service command，而不是只显示原始 slash 文本。
- 可复制输出：现代 TUI 增加只读 JSON 输出区，保存完整 envelope，方便选中复制；摘要区域继续显示人类可读结果。
- slash 解析修复：classic/modern 共用的 `_slash_to_command` 支持重复 `--param KEY=VALUE`，合并为 `params` 字典，避免 `/graph ... --param amazon=1` 被服务层忽略。
- 文档：README / README.zh-CN 补充 64 字符 token 校验、`config set-max-tokens` 和新 TUI 行为说明。

### 风险与后续
