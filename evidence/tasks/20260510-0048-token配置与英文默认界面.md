## 追加记录：TUI 命令输入与预算配置修正

### 任务时间

- 更新时间：2026-05-10
- 当前状态：已完成

### 背景与输入

- 用户反馈：配置过 token 和 max tokens 后不应继续占据大块 TUI 空间；初始焦点应在命令输入；命令输入无效且结果只显示原命令；输出不便复制；输入 `/` 时应像 Codex 一样弹出相关命令。
- 用户要求参考 `cli-creator`：CLI 要有明确 auth/config、稳定 JSON、doctor、能力发现、命令补全和可审计输出。
- 外部依据：公开 Keepa Python API 文档将 access key 描述为 64 character access key string；本轮只做本地形态校验，不承诺服务端有效性。

### 处理过程

- 配置校验：`config set-token` 保存前校验 token 必须为 64 个可见 ASCII 字符、无空白；错误不会写入配置文件。
- 预算配置：新增 `config set-max-tokens`，写入 `max_tokens_per_request`，用于高订阅场景调大单次请求 token 预算提示；接入 service、CLI、capabilities 和 schema snapshot。
- TUI 布局：现代 TUI 改为命令输入优先；启动后焦点固定到 `#command-input`；token/预算输入只在配置不完整时显示，配置完成后折叠为状态行摘要。
- TUI 命令体验：输入 `/` 或前缀时在命令输入上方显示匹配命令；执行后标题显示解析后的 service command，而不是只显示原始 slash 文本。
- 可复制输出：现代 TUI 增加只读 JSON 输出区，保存完整 envelope，方便选中复制；摘要区域继续显示人类可读结果。
- slash 解析修复：classic/modern 共用的 `_slash_to_command` 支持重复 `--param KEY=VALUE`，合并为 `params` 字典，避免 `/graph ... --param amazon=1` 被服务层忽略。
- 文档：README / README.zh-CN 补充 64 字符 token 校验、`config set-max-tokens` 和新 TUI 行为说明。

## 追加记录：从 Textual 面板切换为命令优先 REPL

### 任务时间

- 更新时间：2026-05-10
- 当前状态：已完成

### 背景与输入

- 用户反馈：现有 Textual TUI 看起来“很 AI”，不像官方产品；要求先研究 zread、Codex CLI 或同类 CLI 的真实实现，再决定是否继续使用 Textual。
- 调研依据：
  - 本机 `zread --help` / `zread browse --help` 显示 zread 采用少量顶层命令（`generate`、`browse`、`login`、`config`、`update`、`version`）与 `--stdio` 分离的人机协议，不做面板式首页。
  - OpenAI Codex CLI 官方文档说明：输入 `/` 打开 slash popup 并按命令名过滤；任务运行时可输入 slash command 并按 `Tab` 排队。
  - OpenAI Codex CLI 源码只读调研：TUI 使用 Rust `ratatui`，核心拆分为 `chat_composer`、`footer`、`command_popup`、`chatwidget` 等围绕输入、状态栏、弹窗和转录输出的模块，而不是把终端做成卡片工作台。
  - Python 生态对等方案：`prompt_toolkit` 原生支持 `PromptSession`、history、completer、completion menu、bottom toolbar 和键绑定，更适合当前 Python CLI。

### 处理过程

- 移除现代 TUI 主路径的 Textual 实现，改为 `prompt_toolkit` REPL。
- 保留 `run_modern_tui`、`build_tui_metadata`、`build_command_catalog` 公共入口，降低 CLI 入口改动。
- 现代 TUI 现在采用底部 `kc ›` composer、slash completion、bottom toolbar 状态栏、普通终端输出和完整 JSON envelope。
- 补充 `/token`、`/login`、`/max-tokens`、`/language`、`/config` slash 快捷命令，复用 `config.set-token`、`config.set-max-tokens`、`config.set-language`、`config.show`。
- token 命令转录输出会脱敏，避免 `/token <secret>` 在 REPL 历史输出中泄露；service JSON 仍保持字段级脱敏。
- slash completion 增加轻量 subsequence fuzzy 匹配，靠近 Codex 的命令 popup 手感。
- 测试隔离 `KEEPA_CLI_CONFIG`，避免本机真实配置文件污染 CLI 子进程测试。
- README / README.zh-CN 将 TUI 描述从 Textual 工作台改为 Codex/zread 风格 command-first TUI。
