# tracking 只读写边界评测增强

## 前置说明

- 本轮继续优化调研 Agent 接入安全边界，重点把 `tracking-readonly` 不暴露 tracking 写工具从人工约定升级为 Agent eval 回归项。
- 未访问真实 Keepa API，未消耗真实 token；验证只使用 MCP session、fixture 与本地测试。

## 已落地

1. Agent eval 断言能力增强
   - `keepa_cli.agent_eval` 新增 `not_contains_item` 断言。
   - 可直接断言 list-of-object 不包含某个字段组合，例如 `{"name": "keepa.tracking_add"}`。

2. tracking-readonly 负向 fixture
   - 新增 `tests/agent_eval_fixtures/tracking_readonly_write_boundary.json`。
   - 覆盖 `tools/list {"toolset":"tracking-readonly","profile":"tracking_readonly"}` 不返回：
     - `keepa.tracking_add`
     - `keepa.tracking_remove`
     - `keepa.tracking_remove_all`
     - `keepa.tracking_webhook`
   - 覆盖 `toolset=all + profile=tracking_readonly + allow_tools` 即使显式混入写工具，也只返回活跃的 `keepa.tracking_list`。
   - 继续执行一次 `keepa.tracking_list` fixture 调用，确认只读调用可执行且 budget ledger 无 blocked actions。

## 已执行验证

- `python scripts/check_agent_eval_fixtures.py`：通过，24 specs。

## 风险与后续

- 当前写工具未注册为 MCP tool，负向 fixture 主要防止未来新增写工具后误进入 `tracking-readonly` 发现面。
- 后续若接入 tracking 写 workflow，应新增独立 profile/toolset，并在 fixture 中断言必须返回 `confirmation_required` 或 `inactive_tool`，不能复用 `tracking-readonly`。
