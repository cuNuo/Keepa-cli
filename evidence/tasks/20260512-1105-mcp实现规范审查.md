# MCP 实现规范审查

## 前置说明

- 时间：2026-05-12 11:05 +08:00。
- 范围：生产 MCP stdio 入口 `python -m keepa_cli --mcp`、官方 Python SDK low-level adapter、Streamable HTTP contract fixture、MCP tools/resources/prompts 测试与质量门禁。
- 审查口径：`D:\.codex\skills\mcp-builder\SKILL.md`、MCP 官方 `2025-11-25` tools/transports/lifecycle/resources 规范、项目既有 Serena MCP 记忆。
- 输入来源：本地代码、项目测试、官方公开文档；未访问真实 Keepa API，未消耗真实 token。

## 结论

现有 MCP 实现主体符合当前 MCP `2025-11-25` stdio / tools / resources / prompts 的核心协议形状：`initialize`、`tools/list`、`tools/call`、`resources/list`、`resources/templates/list`、`resources/read`、`prompts/list`、`prompts/get`、`ping` 均有覆盖；tools 暴露 `inputSchema`、`outputSchema`、`structuredContent`、JSON text fallback、annotations 与分页；官方 SDK adapter 和 typed Inspector snapshot 门禁通过。

严格规范层面仍有两个收口项：

1. 工具入参没有完整执行 JSON Schema 类型/枚举/范围校验。`validate_tool_arguments()` 当前只校验未知字段、必填字段和少量跨字段规则；实测 `keepa.categories_search` 的 `term=123` 会被接受并返回成功，而 schema 声明 `term` 是 string。MCP 规范要求 server validate all tool inputs。
2. 生产入口仍是自研 JSON-RPC handler，官方 SDK adapter 仍为隔离入口。当前门禁证明 adapter 与生产输出等价，但长期仍存在 raw handler 与 SDK 类型/协议行为漂移风险。

## 关键证据

- `keepa_cli/agent/mcp.py` 固定 `MCP_PROTOCOL_VERSION = "2025-11-25"`，`initialize` 返回 `serverInfo.name=keepa_mcp`、tools/resources/prompts capabilities。
- `keepa_cli/agent/tools.py` 的 33 个 tools 均包含 `inputSchema`、`outputSchema`、`title`、`annotations.readOnlyHint/destructiveHint/idempotentHint/openWorldHint`。
- 工具标注分布：13 个 read-only/open-world，12 个本地 read-only，6 个本地写产物非破坏性，2 个 cassette promotion destructive。
- 官方 SDK adapter 状态：`mcp==1.27.1` 可用，`production_entrypoint_replaced=false`，SDK stdio 入口为 `python -m keepa_cli.agent.mcp_sdk_adapter --stdio`。
- Streamable HTTP 目前是 boundary contract fixture，不是已落地 HTTP server；不能宣称远程 HTTP transport 已实现。

## 验证

- `python scripts/check_mcp_quality_gate.py --require-sdk --json`：通过，包含 31 个 agent eval fixture、adapter fixture 等价、SDK typed smoke、typed Inspector fixture、SDK Inspector snapshot。
- `python -m unittest tests.test_mcp tests.test_mcp_sdk_adapter tests.test_mcp_http_contract -v`：60 项通过。
- 生产 stdio smoke：initialize 与 `tools/list limit=1` 输出均为 JSON-RPC stdout 消息，未混入非 MCP stdout。
- 代表性 `structuredContent` 对 `outputSchema` 校验：`keepa.context_policy`、`keepa.docs_index`、`keepa.categories_search` 错误路径均通过 schema 校验。

## 后续建议

1. 在 `tools/call` 执行前引入统一 JSON Schema validator，至少覆盖 `type`、`enum`、`minimum`、`oneOf`、`anyOf`、`additionalProperties`，并把失败映射为 `isError=true` 的 `invalid_arguments`。
2. 增加负向 fixture：错误类型、非法 enum、越界 integer、错误 array item 类型，并同时覆盖 raw stdio handler 与 SDK adapter。
3. 等 SDK adapter 负向 schema 校验补齐后，评估把 `python -m keepa_cli --mcp` 切到官方 SDK adapter，或至少给 raw handler 加 JSON-RPC version/lifecycle 严格模式。
4. 若未来要提供远程 MCP，先把现有 Streamable HTTP contract fixture 转成真实 HTTP adapter 集成测试，再实现 Origin、`MCP-Session-Id`、`MCP-Protocol-Version`、notification 202 与 SSE/GET 边界。
