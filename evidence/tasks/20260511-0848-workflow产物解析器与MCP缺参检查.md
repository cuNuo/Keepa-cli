# workflow 产物解析器与 MCP 缺参检查

## 前置说明

- 本轮继续优化 Agent/MCP 长链路执行体验，重点解决 Agent 拿到 `workflow.plan` 后仍需手动拼接 cache key、resource URI、graph path 或 ASIN 列表的问题。
- 未访问真实 Keepa API，未消耗真实 token；验证均使用 fixture、MCP session cache 与本地命令。
- 工作区中已有 tracking-readonly 负向评测改动，本轮确认其与当前 Agent/MCP 安全边界一致，纳入同批收口，不做回退。

## 已落地

1. workflow artifact resolver
   - 新增 `keepa_cli/agent/workflow_resolver.py`。
   - MCP `tools/call` 支持运行时字段：`resource_uri`、`resource_uris`、`artifact`、`artifacts`、`workflow_inputs`、`workflow_context`。
   - 可把同一 session 内的 `keepa://research/{cache_key}`、`keepa://research/{cache_key}/graph`、`keepa://output/{encoded_path}`、本地 JSON path 与 inline artifact 解析成具体 tool 参数；`keepa://graphs/{root}` 保持为图谱来源审计资源，不直接替代完整 graph 输入。
   - 可从上游 payload 推导 ASIN、category id、tracking ASIN 与 research graph，并在成功响应中写入 `data.workflow_resolution` 供 Agent 审计。

2. MCP 缺参检查
   - 在 service 执行前拦截缺少 workflow 依赖的调用。
   - 对 graph merge、brief export、reports build、browse snapshot、products compare/get、categories products/finder-selection、tracking get 等工具返回结构化 `missing_inputs`。
   - `missing_inputs` 会说明缺少字段和可接受来源，避免 Agent 看到泛化的 invalid argument 后继续猜。

3. Agent eval 与 schema
   - 新增 `tests/agent_eval_fixtures/workflow_artifact_resolver_mcp.json`，覆盖 category products -> compare -> graph merge -> reports build 的 resource-first 链路。
   - 新增 `tests/agent_eval_fixtures/workflow_artifact_output_path_mcp.json`，覆盖 `artifact.output.path` -> brief/report 的本地链路。
   - 保留并纳入 `tracking_readonly_write_boundary` 负向 fixture，防止只读 toolset/profile 误暴露 tracking 写工具。
   - Agent eval 当前 26 specs。
   - capabilities schema version 更新为 `2026-05-11.9`，schema snapshot 同步更新。

4. workflow runtime contract 发现入口
   - MCP resources 新增 `keepa://workflow/runtime-contract`，列出支持 `resource_uri`、`resource_uris`、`artifact`、`artifacts`、`workflow_inputs`、`workflow_context` 的 tool、参数名与 `missing_inputs` / `data.workflow_resolution` 约定。
   - `keepa://tools/index`、`keepa://toolsets/{toolset}` 与 `keepa://tools/{name}` 增加 `workflow_runtime` 标记和 contract URI，外部 Agent 不需要加载所有 tool schema 也能安全判断是否可接续上游产物。

5. artifact output path 续接增强
   - `tools/call` 现在可从 `artifact.output.path`、`artifact.data.output.path`、`artifact.raw.output.path` 中提取本地 JSON path。
   - graph merge、brief export、reports build、browse snapshot 可直接消费上一步工具返回的 output path，不要求 Agent 手动转换成 `keepa://output/{encoded_path}`。
   - 新增 `workflow_artifact_output_path_mcp` 评测 fixture，覆盖 artifact path -> brief/report 的纯本地续接链路。

6. 文档与 skill
   - README / README.zh-CN / `docs/agent-contract.md` / `docs/architecture/mcp-agent-tools.md` 已说明 workflow runtime 参数、`workflow_resolution` 与 `missing_inputs`。
   - 项目内 `.codex/skills/keepa-cli` 与 `.codex/skills/keepa-agent-research` 已同步，不写入全局 skill。

## 风险与后续

- 当前 resolver 仍是本地 stdio/MCP 会话内解析，不跨进程持久化 session cache；这符合现阶段“进程内去重、不污染磁盘”的安全边界。
- `keepa://graphs/{root}` 返回的是审计摘要与来源 URI，若要作为 graph merge 输入，应优先使用 `keepa://research/{cache_key}/graph` 或 inline graph artifact。
- 后续最值得做的是把 workflow resolver 的可解析输入 shape 抽成 JSON Schema，并扩展 report/tracking 长链路 eval，覆盖更多 `resource_uri`、`artifact` 与 output path 混合输入。
