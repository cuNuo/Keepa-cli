# TUI 交互收敛优化

## 背景

用户反馈当前 TUI 仍显得冗余、不够像正式 CLI 产品，期望继续参考 Codex CLI / zread 的 command-first 体验：默认聚焦命令输入、slash 补全更自然、输出更克制、配置入口清晰且不占用大空间。

## 本次变更

- 将现代 TUI 启动信息收敛为一行状态与一行操作提示；token 缺失和默认预算只在需要时提示。
- slash 补全改为短命令展示与短描述，选择后插入命令前缀而不是整段示例。
- 默认命令执行只输出转录和摘要，不再每次刷完整 JSON；新增 `/json` 查看上一条完整 envelope。
- 在 TUI catalog 中补齐 `/login`、`/browse`、`/batch`、`/templates`、`/report`、`/cache`、`/cost` 等已有本地 workflow 快捷入口。
- 共享 slash 解析器支持上述 workflow 快捷命令，经典 TUI 与现代 TUI 复用同一路由。
- 为 browse、batch、templates、reports、cache、audit cost 增加短摘要，减少泛化输出。
- token/login 转录继续脱敏；输入解析错误在 REPL 内返回结构化错误摘要，不打断会话。

- 后续如果继续打磨真实视觉，可针对 prompt_toolkit 的 completion menu 配色、toolbar 配色和多行输出间距做一次终端截图验收。

- 若未来需要真正的全屏 TUI，再评估 Rust ratatui 或 Python prompt_toolkit layout；当前阶段继续保持 REPL 更符合用户偏好。
