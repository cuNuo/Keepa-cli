- 做了什么：使用 `coverage run --source=keepa_cli --omit=... -m unittest discover -s tests` 复核后端 scoped coverage。

### 2. 覆盖率配置固化

- 做了什么：在 `pyproject.toml` 增加 `[tool.coverage.run]` 与 `[tool.coverage.report]`。
- 为什么这么做：让后续 `coverage run` 与 `coverage report` 默认使用后端 scoped 口径，并以 `fail_under = 100` 作为硬门禁。
- 结果：配置化命令 `coverage run -m unittest discover -s tests && coverage report` 成功，后端总覆盖率 100%。

### 3. 真实请求链路安全复核

- 做了什么：复核并运行包含 `KeepaClient` fake opener、HTTP 429/5xx retry、gzip、invalid JSON、binary output、POST body、SQLite cache、fixture、dry-run 与 secret redaction 的测试集。
- 为什么这么做：用户特别要求真实请求链路万无一失；项目策略要求默认不消耗真实 token。
- 结果：真实请求链路均由离线 fake/mock/fixture 路径覆盖，未执行真实 Keepa API 请求。
