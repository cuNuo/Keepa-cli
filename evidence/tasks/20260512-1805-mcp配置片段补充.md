# MCP 配置片段补充

## 任务目标

回应用户反馈：README 中没有可直接粘贴到 MCP 客户端的完整 JSON 配置，需要补充开箱可用的配置片段。

## 背景与输入

- 用户输入：希望知道如何使用当前 MCP，并要求提供“直接粘贴就能配置上”的完整 JSON 配置。
- 当前 README 只有 `kc --mcp` 的 JSON-RPC 管道示例，没有 `mcpServers` 级别的客户端配置。
- 当前仓库生产 stdio 入口为 `python -m keepa_cli --mcp` / `kc --mcp`。

## 处理过程

1. 在 `README.md` 的 Agent Mode 中新增完整 JSON 配置：
   - 全局 npm 安装后的 Windows 配置：`cmd /c kc --mcp`。
   - 本仓库本地开发配置：直接调用 `D:\github\Keepa-cli\.venv\Scripts\python.exe -m keepa_cli --mcp`。
2. 在 `README.zh-CN.md` 的 Agent 模式中同步新增中文版本。
3. 明确说明：
   - `env.KEEPA_API_KEY` 可以替换为真实 64 字符 Keepa key。
   - 若已通过 `kc --json config set-token ...` 写入本地配置，可删除 `env` 块。
   - 客户端 server alias 可继续叫 `keepa`，服务端 `serverInfo.name` 是 `keepa_mcp`。

## 验证结果

- README / README.zh-CN 中新增的 JSON code block 均通过 `json.loads` 校验。
- 本地开发入口 `.\.venv\Scripts\python.exe -m keepa_cli --json doctor` 通过。
- 曾验证 `npx -y @cunuo/keepa-cli --json doctor` 在当前 Windows 环境下不可作为直接粘贴推荐配置，会出现 wrapper 找不到底层命令的问题；因此文档改用全局安装后的 `cmd /c kc --mcp` 和本地 `.venv` Python 方案。

## 关联产物

- `README.md`
- `README.zh-CN.md`

## 风险与后续

- `cmd /c kc --mcp` 依赖用户已执行 `npm install -g @cunuo/keepa-cli`，且 `kc` 在 PATH 中。
- 对当前仓库开发者，推荐使用本地 `.venv` Python 配置，不依赖全局 PATH。

## 结论

README 已补充可直接复制的完整 MCP JSON 配置，覆盖 npm 全局安装和本仓库本地开发两种常见用法。
