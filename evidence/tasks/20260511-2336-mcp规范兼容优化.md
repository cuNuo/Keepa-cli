## 处理过程

### 1. 协议与最佳实践差距审计

- 确认已有能力：`tools/list`、`tools/call`、`resources/list`、`resources/templates/list`、`resources/read`、`prompts/list`、`prompts/get`、`structuredContent`、`content[0].text` fallback、`outputSchema`、toolsets/profiles/session cache 均已存在。
- 发现差距：缺少新版协议友好的 `title` metadata；tool annotations 的 `openWorldHint/readOnlyHint/destructiveHint` 对真实 Keepa API 与本地写文件工具不够精确；list 方法没有 cursor 分页；`tools/call` 参数校验失败仍返回 JSON-RPC error，不利于 Agent 按工具错误自修复。

### 2. MCP transport 优化

- 将 MCP 初始化协议版本更新为 `2025-11-25`，`serverInfo.name` 更新为 `keepa_mcp`，同时保留客户端配置中可使用 `keepa` 作为别名的文档说明。
- 增加 `ping` 方法。
- 为 `tools/list`、`resources/list`、`resources/templates/list`、`prompts/list` 增加 cursor 分页；响应包含 `_meta.count`、`_meta.total_count`、`_meta.has_more` 与 `nextCursor`。
- 将工具参数校验失败改为 `tools/call` 工具结果：`isError=true`、`structuredContent.error.kind=invalid_arguments`，并带 `next_action`。

### 3. Tool/resource/prompt metadata 优化

- `ToolDefinition` 增加 `title`、`idempotent`、`open_world` 字段，并输出 top-level `title`、`execution.taskSupport=forbidden`、`annotations.title`。
- 对可能访问真实 Keepa / tracking API 的工具标记 `openWorldHint=true`。
- 对 cassette promotion、report/browse/figure 与 graph/brief export 这类本地写文件或可写 artifact 的工具标记 `readOnlyHint=false`；fixture/evidence promotion 标记 `destructiveHint=true`。
- resources 与 resource templates 自动补 `title`；prompts 自动补 `title`。

### 4. 测试、快照与文档同步

- 更新 `tests/test_mcp.py`，覆盖协议版本、serverInfo、tool title/execution/annotations、cursor 分页、结构化工具错误。
- 更新 `tests/snapshots/agent_schema_snapshot.json`，并统一 LF，避免 `git diff --check` 将 CRLF 判为 trailing whitespace。
- 更新 `docs/agent-contract.md`，说明 MCP 2025-11-25、`keepa_mcp`、cursor 分页、tool metadata、annotation 语义和工具错误边界。

## 风险与后续

- 兼容性风险：`serverInfo.name` 从 `keepa` 调整为 `keepa_mcp`，符合 Python MCP server 命名建议；客户端配置别名仍可使用 `keepa`，但如果外部客户端硬编码 serverInfo.name，需要同步说明。
- 协议风险：本轮仍是手写 JSON-RPC stdio transport，不引入官方 SDK；短期可控，长期需要跟踪 SDK 对 `title`、`execution`、pagination 与 tool error 语义的演进。
- 输出风险：`tools/list all` 仍会返回完整 schema；已有 `limit/cursor`、toolset 和 schema-first resources 可降低上下文压力，后续应在示例中优先演示分页和 `keepa://tools/{name}`。

## 超出本轮的优化建议

1. 增加 MCP Inspector 或协议 fixture 回归：保存 `initialize`、分页 list、工具错误、resources/prompts 的 JSON-RPC 样例，作为独立协议契约测试。
2. 评估官方 Python MCP SDK 适配层：先保持 `run_command` 核心不变，只在外层提供 SDK server adapter，用于对比 Inspector、Claude Desktop、Codex MCP client 兼容性。
3. 增加 MCP client migration guide：明确 `serverInfo.name=keepa_mcp` 与客户端别名 `keepa` 的区别，并给出旧配置无痛迁移路径。
4. 将 `tools/list all` 的推荐链路改为 resource-first：Agent 先读 `keepa://tools/index`，再按需读 `keepa://tools/{name}`，最后才调用全量 `tools/list`。
