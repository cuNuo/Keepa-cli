# MCP 调研 Agent 最佳实践落地

## 任务目标

- 根据上一轮 MCP 设计调研结果，逐项优化 Keepa-cli MCP。
- 优先面向未来调研 Agent 接入，落实 policy/status、target resolution、context query、工具过滤、prompt、skill、文档与评测。
- 保持 offline-first，不访问真实 Keepa API，不消耗真实 token。

## 背景与输入

- 用户要求：根据调研结果认真优化，一项接一项，确保最佳实践。
- 用户约束：skill 只能放在项目内，不写全局路径。
- 参考 evidence：`evidence/tasks/20260511-0001-mcp设计调研与增进方案.md`。

## 已落地改动

1. 新增 `keepa_cli/research_context.py`
   - `build_context_policy()`：输出 offline-first policy、tool gating、live Keepa 安全状态、read/write roots、never_commit。
   - `resolve_research_target()`：本地解析 ASIN、UPC/EAN、seller、category、fixture、evidence、keyword 候选。
   - `query_research_context()`：基于目标列出本地 policy、tools、evidence、ASIN fixture 等资源。

2. MCP resources 与 service command
   - 新增 `keepa://context/policy` resource。
   - 新增 service commands：
     - `context.policy`
     - `research.target.resolve`
     - `research.context.query`

3. MCP tools 与工具过滤
   - 新增 tools：
     - `keepa.context_policy`
     - `keepa.resolve_research_target`
     - `keepa.query_research_context`
   - `tools/list` 新增 `allow_tools` / `exclude_tools`，支持按工作流收窄 schema。

4. CLI 入口
   - 新增 `docs index/read`。
   - 新增 `research policy`、`research resolve-target`、`research context`。

5. Prompts、skill 与文档
   - 新增 MCP prompt：`keepa.research_agent_start`。
   - 更新项目内 skills：
     - `.codex/skills/keepa-agent-research/SKILL.md`
     - `.codex/skills/keepa-cli/SKILL.md`
   - 更新 README、`docs/agent-contract.md`、`docs/architecture/mcp-agent-tools.md`。

6. 评测与快照
   - 新增 Agent eval specs：
     - `tests/agent_eval_fixtures/research_context_resolution.json`
     - `tests/agent_eval_fixtures/research_context_query.json`
     - `tests/agent_eval_fixtures/reports_build_research_graph_output.json`
     - `tests/agent_eval_fixtures/research_context_policy_and_target.json`
   - 更新 `tests/snapshots/agent_schema_snapshot.json`。
   - 补充 service、MCP、capabilities 单测。

7. 本轮继续增强
   - 新增 `keepa://research/{cache_key}`，在同一 MCP session 内按 cache key 审计 cached result、budget ledger、evidence、provenance 和 research graph summary。
   - 新增 `keepa://graphs/{root}`，按 research graph root 从 session cache 与本地 fixture 查图谱来源。
   - 新增 `keepa://research/{cache_key}/brief` 与 `keepa://research/{cache_key}/graph`，让 Agent 可按同一缓存结果分别读取 research brief 与 graph 摘要。
   - `reports.build` 可直接消费 `research_graph.merge --out` 的 merged graph JSON；Markdown 生成实体与关系表，JSON 输出增加 `research_graph_report`。
   - GitHub Pages `docs/index.html` 重做为更接近成熟开源文档站的开发者入口：明确 quickstart、docs、Agent resources、MCP URI 与安全状态。
   - `.github/workflows/pages.yml` 部署后增加非阻断 Pages URL 健康检查，防止公开文档入口静默失效。
   - 通过 Playwright 本地检查桌面与 390px 移动端布局，修复 quickstart 区域横向溢出，并增加内联 favicon，避免健康检查和控制台被 favicon 404 噪声污染。
