# Cache 审计与命令族拆分任务日志

## 任务目标

- 给常用 live 命令显式补齐 `--cache-ttl` / `--no-cache`，不只依赖环境变量控制缓存。
- 增加 `cache inspect` 与 `cache prune-expired`，支持 Agent 审计单条 cache key 与清理过期缓存。
- 继续按 command family 拆分 `products`、`categories`、`tracking`，降低 `keepa_cli/cli.py` 与 `keepa_cli/service.py` 复杂度。

## 背景与输入

- 用户要求在 Phase 3 SQLite response cache 已落地后继续补审计与清理入口，并推进结构性降债。
- 本轮默认不访问真实 Keepa API，不消耗真实 token；live 行为通过 fake opener、fixture、dry-run 与 schema 测试验证。
- 当前工作区已有多项前序 Agent/MCP 与 research graph 改动，本轮只围绕 cache 控制、cache 审计命令、products/categories/tracking family 拆分和交付收口处理，不回退无关改动。

## 处理过程

- `KeepaClient.request(...)` 增加 `cache_ttl_seconds` 与 `no_cache` 入参，支持调用方覆盖缓存 TTL 或强制绕过缓存。
- `SQLiteResponseCache` 增加 `inspect()` 与 `prune_expired()`，让 cache stats/clear 之外具备单 key 审计和过期项清理能力。
- CLI 与 service 接入 `cache inspect <cache_key>`、`cache prune-expired [--dry-run] [--cache-path]`。
- 常用 live JSON 命令显式接入 `--cache-ttl` 与 `--no-cache`，覆盖 products、categories、tracking、history、finder、deals、sellers、bestsellers、topsellers、tokens、lightningdeals、raw request 等入口。
- 新增并接入 command family 模块：
  - `keepa_cli/cli_builders/products.py`
  - `keepa_cli/cli_builders/categories.py`
  - `keepa_cli/cli_builders/tracking.py`
  - `keepa_cli/commands/products.py`
  - `keepa_cli/commands/categories.py`
  - `keepa_cli/commands/tracking.py`
- 更新 capabilities、Agent contract、README 与中英文文档，说明显式 cache 控制、cache inspect/prune-expired 与 Agent 审计用途。
- 更新测试覆盖 cache、client、CLI、capabilities 与 schema snapshot。

## 风险与后续

- 未执行真实 Keepa live 请求；真实网络路径仍需在用户显式授权并配置 `KEEPA_API_KEY` 后做低成本 smoke。
- `cli.py` 与 `service.py` 已拆出 products/categories/tracking，但仍可继续按 raw、history、finder、deals 等 family 降低入口复杂度。
- 可进一步增加 cache key explain/hash 辅助命令，把请求参数到 cache key 的映射变为可审计输出。
- 可把 live cache policy 纳入 schema 或 release gate 的固定检查，防止新增 live 命令遗漏 `--cache-ttl` / `--no-cache`。
