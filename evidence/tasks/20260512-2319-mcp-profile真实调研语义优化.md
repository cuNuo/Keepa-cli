# MCP profile 与真实调研语义优化

## 背景

用户反馈 Keepa MCP 的 `offline_fixture_only` / `dry_run_default` profile 行为与 skill 文档不一致：研究工具在安全 profile 下不可发现或直接 inactive，导致 fixture / dry-run 调研无法按文档执行；同时 `products_compare` 大结果容易让 MCP 客户端截断，`categories.products` 高成本提示需要清楚说明官方 50 token 成本，报告 brief 需要承接外部广告、专利/IP 与宣称风险证据。

## 变更摘要

- `offline_fixture_only` 与 `dry_run_default` 保持产品、类目、Finder、Deals、榜单等研究工具可发现。
- MCP `tools/call` 对安全 profile 增加执行前 guard：未传 `fixture` 或 `dry_run` 时返回 `profile_requires_fixture_or_dry_run`，并提示真实调研切换到 `live_read_allowed`。
- `products_compare` 的 MCP `structuredContent` 默认压缩为 decision summary rows，并通过 `keepa://research/{cache_key}` 暴露同会话完整 payload；CLI JSON 行为保持完整输出。
- `categories.products` 保留官方 `/bestsellers` 50 token 事实，不伪造低成本 scout；确认提示和 dry-run 输出增加替代路径：Finder scaffold、category search、已知 ASIN compare。
- `research_brief.export` 新增 `external_signal_stub`、`ip_risk_inputs`、`claim_risk_inputs` slot；`reports.build` / `research_brief_export` 支持 `out_dir` 产出可提交摘要。
- README、中文 README、Agent contract、MCP architecture 文档与 Keepa skills 同步移除“真实调研离线优先”表述，明确 fixture/dry-run 只用于测试开发、回归与预算预估。

## 风险与后续

- `products_compare` 的 MCP structuredContent 被有意压缩；依赖完整 rows 的 Agent 需要读取 `keepa://research/{cache_key}`。CLI 与 service 原始 payload 未压缩。
- `live_read_allowed` 已明确用于真实调研，但高成本请求仍依赖既有 `confirmation_required` / `yes=true` 机制，避免无确认消耗 token。
- `research_brief` 只提供外部信号/IP/claim slot，不直接做广告或专利联网检索；外部证据仍由 web/Exa/人工 FTO 流程合并。
