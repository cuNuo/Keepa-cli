# workflow 输入产物资源契约增强

## 前置说明

- 本轮继续优化 Agent/MCP 接入体验，重点解决 `workflow.plan` 里 Agent 仍需从 CLI 字符串推断中间输入和输出的问题。
- 未访问真实 Keepa API，未消耗真实 token；验证均使用本地 service、fixture、dry-run、MCP session 与发布门禁。
- 工作区中发现的 resource-first MCP session eval 变更与本轮目标一致，已纳入审查、验证和收口。

## 已落地

1. `workflow.plan` 增加结构化执行契约
   - 顶层新增 `workflow_inputs`：明确用户参数、后续步骤产物和占位值来源。
   - 顶层新增 `artifacts`：列出中间产物和最终产物的 kind、producer、path 与可用 resource template。
   - 顶层新增 `resource_templates`：汇总 `keepa://workflow/{encoded_params}/policy`、`keepa://research/{cache_key}`、`keepa://graphs/{root}`、`keepa://output/{encoded_path}` 等可读入口。
   - 每个 step 新增 `input_refs` 与 `artifact_refs`，让 Agent 用字段连接步骤，不再解析 `cli` 字符串。

2. workflow policy resource 同步增强
   - `keepa://workflow/{encoded_params}/policy` 现在也返回 `workflow_inputs`、`artifacts`、`resource_templates`。
   - `step_summary` 增加 `input_refs` 与 `artifact_refs`。
   - `recommended_read_order` 更新为 `workflow_inputs -> artifacts -> resource_templates -> workflow_policy -> step_summary -> totals`。

3. Agent evaluation 增强
   - `keepa_cli.agent_eval` 新增 `mcp_session` fixture kind，可在同一个 `AgentSession` 内连续执行 `resources/read`、`tools/list`、`tools/call`。
   - 新增 `resource_first_research_agent_start.json`，固化资源优先链路：context policy -> resolve target -> workflow policy resource -> filtered tools/list -> fixture execute。
   - Agent eval specs 从 21 扩展到 22。

4. 文档、skill 与 schema 同步
   - README / 中文 README、`docs/agent-contract.md`、`docs/architecture/mcp-agent-tools.md`、项目内 `keepa-cli` / `keepa-agent-research` skills 均补充 `workflow_inputs` / `artifacts` / refs 的使用规则。
   - capabilities schema version 更新到 `2026-05-11.7`。
   - 刷新 `tests/snapshots/agent_schema_snapshot.json`，并保持 LF 行尾。

## 风险与后续

- `workflow_inputs` / `artifacts` 当前是静态契约，后续可以在 AgentSession 内用真实 `cache_key` 自动回填 `artifact.resource_uri`。
- `report-research` 仍保留 `<CATEGORY_JSON>` 等占位输入，适合规划阶段；后续可新增 `--from-cache` / `--graph-root` 参数，让计划生成时直接绑定资源。
- 下一步最适合继续完善：实现 workflow artifact resolver，把 `keepa://research/{cache_key}`、`keepa://graphs/{root}` 与本地 path 统一解析为 step 参数，并在 MCP tools/call 前做输入完整性检查。
